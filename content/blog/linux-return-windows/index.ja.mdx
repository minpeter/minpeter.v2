## 私がLinuxに行くことにした理由

Windows 11にアップデートしてからだいぶ経った私は... そろそろ新しさを求めていて、おまけに2コア4スレッドのi5-6300uの限界を感じてLinuxへの移行を決めた。
しかしこれが間違った選択だと気づくまでそう長くはかからなかった...

---

## Linuxデスクトップ環境を設定しながら感じたこと

1. ハングル入力の難しさ
   難しい。特にWayland環境でのハングル入力は初期段階で、実際Waylandの入力メソッド標準がまだないのでArchのWMにSwayを決めたことをすごく後悔した。
   そのためハングル入力だけでも解決しようとFedoraをインストールしたら、新しくアップデートされたGNOME 40が迎えてくれた。
   当然Waylandがデフォルトで設定されていて、現在Waylandで使用可能な数少ない入力メソッドであるiBusもデフォルトだったので、難なくハングルを入力できた。
   ここでもう一つ問題が発生する...

2. Debian（deb）系列でなければ冷遇されるかもしれないパッケージ環境...(?)
   Fedoraをインストールしてハングル入力がうまくいって喜んでいたところだった。Discordをインストールしようとしたら公式サポートパッケージが\*.debだけだった...
   残りは圧縮ファイル形式の実行ファイルだけくれたんだけど... 現実を突きつけられた...
   パッケージインストールが簡単だからLinuxを好む部分も確かにあるのに、なぜ私はこんなことをしているのか...;;
   もちろんArchを使えば最強のAURが全部解決してくれたが、私はSwayでのハングル入力問題でArchは諦めた状態...

3. 均一でないウィンドウ
   これは実はWindowsも少し該当するが、ウィンドウの見た目がバラバラな子たちがいる。
   WindowsではレガシープログラムWaylandの入力メソッド標準がまだないのでArchのWMにSwayを決めたことをすごく後悔した。で発生するが、LinuxではQTやGTKのようなツールキットの違いから来ると聞いた。
   分からない、まだ初心者だから解決できる問題かもしれない。
   でも機能的に問題があるわけではないので、ただのWindowsへの郷愁病と言える。

4. 設定していると来る心身衰弱
   DockerとMinikubeをインストールして使ってみたら、確かにネイティブな味が感じられた。
   たくさん開いていてもRAMを4GBしか占有しない魔法！（OS含む！）
   私が望んでいたのはここまでなのに、インストールしながら精神的な苦痛までは望んでいなかった。
   正直Linuxをうまく扱えれば関係ないが、6ヶ月ほどWindows使ってから戻ろうとしたら頭がくらくらする。
   前にLinux初めて触ったときは珍しかったし、時間がすごーく余っていたから直接いじるのも良かった。
   でも今は3日間大騒ぎしたら精神も疲れて、開発もできないのが（ブログ記事さえも...）もっと大きく感じられた。
